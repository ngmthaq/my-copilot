# Vite Code Splitting

## 1. How It Works

Vite uses Rollup under the hood. Every dynamic `import()` automatically creates a separate chunk. The goal is to load only what's needed on each page.

---

## 2. Dynamic Imports (Lazy Loading)

```typescript
// Instead of:
import HeavyComponent from "./HeavyComponent";

// Use:
const HeavyComponent = React.lazy(() => import("./HeavyComponent"));
```

Generic dynamic import:

```typescript
const module = await import("./utils/chartHelpers");
module.drawChart(data);
```

---

## 3. Lazy-Loading Routes (TanStack Router Example)

TanStack Router has built-in support for lazy routes via the `lazy()` option on a route definition. No `React.lazy` or `Suspense` needed.

```typescript
// src/routes/__root.tsx
import { createRootRoute, Outlet } from '@tanstack/react-router';

export const Route = createRootRoute({
  component: () => <Outlet />,
});
```

```typescript
// src/routes/index.tsx  (eagerly loaded — small, so it's fine)
import { createFileRoute } from '@tanstack/react-router';

export const Route = createFileRoute('/')({ component: HomePage });
function HomePage() { return <div>Home</div>; }
```

```typescript
// src/routes/dashboard.lazy.tsx  (actual heavy component)
export function DashboardPage() {
  return <div>Dashboard</div>;
}
```

```typescript
// src/main.tsx
import { RouterProvider, createRouter } from '@tanstack/react-router';
import { routeTree } from './routeTree.gen'; // generated by TanStack Router Vite plugin

const router = createRouter({ routeTree });

ReactDOM.createRoot(document.getElementById('root')!).render(
  <RouterProvider router={router} />
);
```

> Use the `@tanstack/router-plugin` Vite plugin to auto-generate `routeTree.gen.ts` from your file-based routes.

## 4. manualChunks — Control Chunk Grouping

Group specific libraries into named chunks. This prevents them from being bundled awkwardly.

```typescript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          // put all React packages into one vendor chunk
          "vendor-react": ["react", "react-dom", "@tanstack/react-router"],
          // put UI libraries into their own chunk
          "vendor-ui": ["@mui/material", "@emotion/react", "@emotion/styled"],
          // utility libraries
          "vendor-utils": ["lodash-es", "date-fns", "axios"],
        },
      },
    },
  },
});
```

---

## 5. manualChunks with a Function

For more control, use a function:

```typescript
build: {
  rollupOptions: {
    output: {
      manualChunks(id) {
        if (id.includes('node_modules')) {
          // put all node_modules into a single 'vendor' chunk
          return 'vendor';
        }
      },
    },
  },
},
```

---

## 6. Preloading Chunks

Vite automatically adds `<link rel="modulepreload">` for chunks needed on the initial page. For other chunks, trigger preload manually:

```typescript
// Preload a chunk when user hovers a button (before they click)
const handleMouseEnter = () => {
  import("./pages/HeavyPage"); // triggers download but doesn't render
};
```
